# 아이 치과 진료 기록 관리 웹앱 (치아노트 유사 프로젝트) 기획 브리핑

## 1. UX 디자이너 (치아 지도 UI)
- 직관적인 유치/영구치 전환 및 치식 번호 UI 구성
- 부모가 쉽게 이해하고 기록할 수 있도록 시각적인 3D/2D 치아 그래픽 제공
- 충치, 발치, 신경치료 등 상태별 직관적인 컬러 및 아이콘 시스템 도입

## 2. 백엔드 개발자 (DB 구조)
- 환자(아이), 보호자(부모), 치아 상태 기록, 진료 이력 엔티티 간의 관계형 DB 설계 (PostgreSQL 권장)
- 이미지(X-ray, 구강 사진) 저장을 위한 클라우드 스토리지(S3) 매핑 및 메타데이터 관리
- 시계열 데이터를 활용한 치아 성장 및 진료 히스토리 추적 테이블 구성

## 3. 비즈니스 분석가 (차별화 전략)
- 기존 앱들과의 차별화를 위해 '성장별 맞춤형 구강 관리 리포트' 제공
- 소아치과 예약 연동 및 위치 기반 추천 서비스 기획
- 양치질 습관 트래킹 및 게이미피케이션(리워드/뱃지) 요소 도입으로 유저 리텐션 강화

## 4. 보안 전문가 (의료법 준수)
- 개인 민감 정보(의료 데이터, 생년월일 등)의 암호화 저장 및 전송 (HIPAA 및 국내 개인정보보호법, 의료법 준수)
- 접근 통제 및 본인 인증 강화, 보호자-자녀 데이터 접근 권한 분리
- 데이터 백업 및 복구 계획 수립

## 5. 프론트엔드 개발자 (Next.js 스택)
- SEO와 초기 로딩 속도 최적화를 위한 Next.js App Router 및 SSR/SSG 활용
- 상태 관리를 위한 Zustand 또는 React Query 도입 (오프라인 캐싱 고려)
- 다양한 디바이스(모바일 뷰어 중심)를 고려한 반응형 PWA(Progressive Web App) 구축
